# All She Needs – Project Management App

A modern, cross-platform project management app built with **Flutter**, **Firebase**, and **Supabase**. Designed for small businesses and teams to efficiently manage clients, vendors, and orders, with beautiful UI, robust authentication, image storage, and multi-language support (English & Arabic).

---

## 🚀 Features

- **Client, Vendor, and Order Management**
- **Authentication**: Email/Password & Google Sign-In (Firebase Auth)
- **Cloud Firestore**: Real-time data storage for users, clients, vendors, and orders
- **Supabase Storage**: Upload and manage order/client images
- **Image Picker & Gallery**: Attach and view images for orders
- **Financial Dashboard**: Track sales, purchases, charges, and net profit
- **Multi-language**: English & Arabic, with easy language switching
- **Modern Theming**: Light/Dark mode, gradients, and custom themes
- **Responsive UI**: Works on mobile, tablet, and desktop
- **State Management**: Bloc & Provider
- **Comprehensive Error Handling**

## ⚙️ Setup & Configuration

### 1. Clone the Repository
```bash
git clone <https://github.com/MohamedElbagoury11/Orders-Management.git>
cd projectmange
```

### 2. Install Dependencies
```bash
flutter pub get
```

### 3. Firebase Setup
- Create a Firebase project.
- Add Android/iOS/web apps in Firebase console.
- Download `google-services.json` (Android) and `GoogleService-Info.plist` (iOS) and place them in the respective platform folders.
- Configure Firebase options in `lib/firebase_options.dart` (auto-generated by `flutterfire configure`).
- See [GOOGLE_SIGNIN_SETUP.md](GOOGLE_SIGNIN_SETUP.md) for Google Sign-In troubleshooting and setup.

### 4. Supabase Setup (Image Storage)
- Create a Supabase project and a public storage bucket named `order-images`.
- Set up storage policies for public read and authenticated uploads.
- Add your Supabase URL and anon key in `main.dart`:
  ```dart
  await Supabase.initialize(
    url: 'YOUR_SUPABASE_URL',
    anonKey: 'YOUR_SUPABASE_ANON_KEY',
  );
  ```
- See [SUPABASE_BUCKET_SETUP.md](SUPABASE_BUCKET_SETUP.md) and [SUPABASE_IMAGE_SETUP.md](SUPABASE_IMAGE_SETUP.md) for detailed instructions.

### 5. Assets & Icons
- App icons and images are in `assets/icon/` and `assets/images/`.
- Configure launcher icons with `flutter_launcher_icons`:
  ```bash
  flutter pub run flutter_launcher_icons:main
  ```

### 6. Environment Variables
- Sensitive keys (Firebase, Supabase) are stored in code for demo; use secure storage for production.
